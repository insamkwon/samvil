## Runtime enforcement

The v3.2 ledger (①) records `claimed_by` and `verified_by` for every
claim. The `validate_role_separation` MCP tool then checks:

1. `claimed_by != verified_by` (identity).
2. `verified_by` resolves to a Judge role (or an out-of-band verifier
   like `agent:user` or `agent:orchestrator-agent`).
3. `claimed_by` is not a Judge for its own subject.

Violations are rejected by the ledger, not just the tool — a skill cannot
bypass the check by writing directly to the JSONL file and still produce
a `status: verified` row without the Judge identity set correctly.

## Adding a new agent

1. Add the agent file under `agents/`.
2. Register the primary role in
   `mcp/samvil_mcp/model_role.DEFAULT_ROLES` (or in `OUT_OF_BAND` if
   it's not one of the six).
3. Run `python3 scripts/apply-role-tags.py` to inject the frontmatter.
4. Run `python3 scripts/render-role-inventory.py` to refresh this file.
5. Run the test suite.
