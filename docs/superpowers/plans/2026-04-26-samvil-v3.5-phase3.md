# SAMVIL v3.5 Phase 3 — Telemetry + Run Observability

**Date:** 2026-04-26
**Base:** v3.4.0 Multi-Host Runtime + Pattern Registry
**Target:** v3.5.0

## Goal

Turn SAMVIL's existing files and MCP traces into an operator-readable run
telemetry layer. Phase 3 should answer: what happened, where did it slow down,
what failed, what is blocked, and what should a future retro learn?

## Phase 3 Outputs

| Area | Output |
|---|---|
| Run report | `.samvil/run-report.json` + MCP build/read/render tools |
| Event taxonomy | normalized stage event categories and failure/retry counters |
| Health rollup | MCP health failures grouped by tool and recent failure |
| Retro feed | telemetry observations promoted into retro candidates |
| Status surface | `samvil-status` / report rendering shows one-screen run state |

### W2 — Event Taxonomy + Retry Counters

- [ ] Categorize stage events into start, complete, fail, retry, blocked, skip.
- [ ] Add per-stage durations when timestamps exist.
- [ ] Track retry/fix loops and failure signatures.
- [ ] Acceptance: a run with build fail + fix + pass renders a coherent timeline.

### W3 — Retro Observation Feed

- [ ] Convert report findings into retro candidate observations.
- [ ] Deduplicate recurring failures by tool/stage/signature.
- [ ] Add MCP tool or script to append selected observations.
- [ ] Acceptance: telemetry can propose at least one actionable retro row.

### W4 — Status Surface Integration

- [ ] Teach `scripts/samvil-status.py` to read `.samvil/run-report.json`.
- [ ] Add a concise human report for non-engineer status inspection.
- [ ] Acceptance: one command shows stage, gates, health, continuation, and next action.

### W5 — Phase 3 Dogfood + Release

- [ ] Run telemetry snapshot against synthetic and live-ish project fixtures.
- [ ] Record discovered blind spots.
- [ ] Release as `v3.5.0`.

## Non-Goals

- External analytics service.
- LLM-based telemetry summaries in W1.
- Replacing Claim Ledger or EventStore.
- Real-time dashboard UI.

## Release Bar

- Full test suite green.
- Pre-commit check green.
- Run report tools are deterministic and file-only.
- Status surface can explain the latest run without reading raw JSONL.
