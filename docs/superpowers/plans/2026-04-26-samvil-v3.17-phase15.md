# SAMVIL v3.17 Phase 15 — External CI Mirror

**Date:** 2026-04-26
**Base:** v3.16.0 Release Evidence Bundle
**Target:** v3.17.1

> Patch note: the initial v3.17.0 remote artifact proved the workflow could
> upload blocked release evidence while the job stayed green because `tee`
> masked the runner exit code. v3.17.1 closes that gap with explicit Playwright
> browser installation and `pipefail` on piped release commands.

## Goal

Mirror the local release checks in GitHub Actions so release readiness is no
longer only local-session evidence. Phase 15 runs the default release runner in
CI, builds the release evidence bundle, and uploads the machine + markdown
artifacts for PR/main review.

## Phase 15 Outputs

| Area | Output |
|---|---|
| CI | `.github/workflows/release-checks.yml` |
| Runner | `scripts/run-release-checks.py --format json` |
| Bundle | `scripts/build-release-bundle.py --format json` |
| Artifacts | `samvil-release-evidence` upload |
| Validator | `scripts/validate-ci-workflow.py` |
| Tests | workflow contract pytest coverage |
| Release | `v3.17.1` |

## Non-Goals

- Publishing to package registries.
- Replacing the local release runner.
- Adding branch protection rules.

## Release Bar

- Workflow mirrors local release runner and evidence bundle commands.
- Workflow artifacts preserve report, summary, and command JSON.
- Local workflow validator passes.
- Full release runner and pre-commit checks pass.
- Main push or manual dispatch produces a GitHub Actions run, unless blocked by
  repository authentication or provider availability.
