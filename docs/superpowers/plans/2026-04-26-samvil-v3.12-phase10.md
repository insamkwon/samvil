# SAMVIL v3.12 Phase 10 — Inspection Repair Execution Loop

**Date:** 2026-04-26
**Base:** v3.11.0 Inspection Feedback Loop
**Target:** v3.12.0

## Goal

Close the first create -> inspect -> fail -> repair -> reinspect loop. Phase 10
turns failed inspection reports into repair plans, records before/after repair
evidence, and verifies that the repaired app passes inspection.

## Phase 10 Outputs

| Area | Output |
|---|---|
| Repair module | `mcp/samvil_mcp/repair.py` |
| Repair artifacts | `.samvil/repair-plan.json`, `.samvil/repair-report.json` |
| MCP tools | build/read/render repair plan and repair report |
| Status surface | repair status and next action |
| Dogfood script | `scripts/phase10-inspection-repair-dogfood.py` |
| Release | `v3.12.0` |

## Workstreams

### W1 — Repair Plan Contract

- [x] Convert inspection failures into ordered repair actions.
- [x] Include target, instruction, verification, severity, and status.
- [x] Acceptance: deterministic plan is unit-tested.

### W2 — Repair Report Contract

- [x] Compare before/after inspection reports.
- [x] Record resolved and remaining failures.
- [x] Mark repair as verified only when after inspection passes.
- [x] Acceptance: before fail -> after pass is proven in report.

- [x] Add build/read/render tools for repair plan/report.
