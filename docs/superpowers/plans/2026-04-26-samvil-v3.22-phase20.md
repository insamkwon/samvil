# SAMVIL v3.22 Phase 20 — QA Materialization

**Date:** 2026-04-26
**Base:** v3.21.0 QA Synthesis Gate
**Target:** v3.22.0

## Goal

Turn the central QA synthesis decision into durable SAMVIL run artifacts.
Phase 19 produced the deterministic verdict; Phase 20 writes that verdict into
`.samvil/qa-results.json`, `.samvil/qa-report.md`, `.samvil/events.jsonl`, and
`project.state.json`, then exposes it through run reports and `samvil-status`.

## Phase 20 Outputs

| Area | Output |
|---|---|
| Materialization core | `materialize_qa_synthesis` in `mcp/samvil_mcp/qa_synthesis.py` |
| MCP tool | `materialize_qa_synthesis` |
| Status surface | `scripts/samvil-status.py` QA panel and next action |
| Telemetry | `build_run_report(...).qa` summary |
| Dogfood | `scripts/phase20-qa-materialization-dogfood.py` |
| Tests | QA synthesis, status, MCP smoke, Phase 20 dogfood |
| Release | `v3.22.0` |

- Allowing independent agents to write materialized state.
- Changing minimal tier inline QA behavior.
- Replacing browser evidence collection.
