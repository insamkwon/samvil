# SAMVIL v3.24 Phase 22 — QA Recovery Routing

## Goal

Phase 21 can detect a blocked QA revise loop. Phase 22 turns that blocked
state into a concrete continuation route so the host can move to evolve,
targeted repair, or retro without relying on free-text instructions.

## Phase 22 Outputs

| Area | Output |
| --- | --- |
| Core | `mcp/samvil_mcp/qa_routing.py` |
| Artifacts | `.samvil/qa-routing.json`, `.samvil/next-skill.json` |
| MCP | `build_qa_recovery_routing`, `materialize_qa_recovery_routing` |
| Runtime | `build_run_report` and `samvil-status` expose the selected route |
| Dogfood | `scripts/phase22-qa-recovery-routing-dogfood.py` |
| Tests | QA routing, telemetry/status, MCP smoke, Phase 22 dogfood |
