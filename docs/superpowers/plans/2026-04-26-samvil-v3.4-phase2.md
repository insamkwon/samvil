# SAMVIL v3.4 Phase 2 — Multi-Host Skill Migration + Pattern Registry

**Date:** 2026-04-26
**Base:** v3.3.0 4-Layer Portability Foundation
**Target:** v3.4.0

## Goal

Turn the v3.3 skeleton into a practical multi-host SAMVIL runtime:

1. migrate the remaining high-traffic skills to ultra-thin MCP-driven bodies
2. add real Codex/OpenCode continuation adapters
3. enrich Codebase Manifest with pattern/dependency intelligence
4. add a regression suite that proves the same seed can progress across hosts

## Phase 2 Outputs

| Area | Output |
|---|---|
| Skill migration | `samvil-interview`, `samvil-design`, `samvil-build`, `samvil-qa` ultra-thin PoCs |
| Host adapters | Codex/OpenCode file-marker resume recipes and smoke scripts |
| Pattern Registry | `mcp/samvil_mcp/pattern_registry.py` + references schema |
| Manifest v2 | static import dependencies + module summaries |
| Regression suite | cross-host replay fixtures under `mcp/tests/fixtures/phase2/` |

## Workstreams

### W1 — Skill Migration Harness

- Define a repeatable migration checklist from `samvil-seed`.
- Create `scripts/skill-thinness-report.py` to track active vs legacy line counts.
- Migrate `samvil-design` first because it benefits most from Orchestrator.
- Acceptance: active design skill under 120 lines, legacy preserved, skill wiring PASS.

### W2 — Codex/OpenCode Continuation

- Standardize `.samvil/next-skill.json` shape.
- Add `references/host-continuation.md`.
- Add smoke script that reads a marker and prints the next action for Codex CLI.
- Acceptance: file-marker chain from seed -> design and seed -> council is reproducible without Claude Skill tool.

### W3 — Pattern Registry

- Add registry of framework/domain patterns: Next.js app-router, Vite, Phaser, Expo, dashboard.
- Expose MCP read/render tools.
- Connect Build/QA prompts to pattern IDs, not copied paragraphs.
- Acceptance: registry lookup works without scanning long skill bodies.

### W4 — Manifest v2 Intelligence

- Add import graph extraction for TypeScript/JavaScript/Python.
- Add module summaries with explicit `generated_by` and timestamp metadata.
- Add confidence tags for regex/AST/manual sources.
- Acceptance: real Next.js smoke shows dependencies among at least 3 modules.

### W5 — Cross-Host Regression Suite

- Capture one small web-app seed fixture.
- Replay interview-summary -> seed -> design handoff across Claude-style and file-marker style hosts.
- Add pre-commit smoke for host adapter regressions.
- Acceptance: same seed reaches the same next-stage marker under both host modes.

## Non-Goals

- Full replacement of all 15 skills in one release.
- Real remote Codex/OpenCode process orchestration.
- Semantic code search beyond explicit manifest/pattern fields.
- Breaking seed schema changes.

## Release Bar

- Full test suite green.
- Pre-commit check green.
- At least two active skills are ultra-thin and host-aware.
- Codex-style file-marker continuation documented and smoke-tested.
- v3.4.0 changelog records which skills remain legacy.
