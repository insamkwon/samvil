# SAMVIL v3.23 Phase 21 — QA Convergence Gate

## Goal

Phase 20 persisted the QA synthesis verdict. Phase 21 turns repeated QA
iterations into a deterministic convergence gate so SAMVIL can tell the
difference between a productive revise loop and a blocked loop that needs
manual intervention or seed evolution.

## Phase 21 Outputs

| Area | Output |
| --- | --- |
| Core | `evaluate_qa_convergence` gate in `qa_synthesis.py` |
| Artifacts | convergence embedded in `.samvil/qa-results.json` and `qa_history` |
| Events | `qa_blocked` / `qa_failed` event drafts from materialization |
| Runtime | `build_run_report` and `samvil-status` prioritize blocked convergence |
| Dogfood | `scripts/phase21-qa-convergence-gate-dogfood.py` |
| Tests | QA synthesis, status, MCP smoke, Phase 21 dogfood |

## Acceptance

- A first REVISE with fixable issues remains `continue`.
- A second REVISE with the same issue IDs becomes `blocked`.
- Status and run report recommend manual intervention before another blind
  auto-fix attempt.
