# SAMVIL v3.19 Phase 17 — Verified Release Publisher

**Date:** 2026-04-26
**Base:** v3.18.0 Remote Release Gate
**Target:** v3.19.0

## Goal

Turn the v3.18 remote release gate into a publish workflow. Phase 17 provides a
single command that pushes the release branch, waits for GitHub Actions, checks
the uploaded release evidence artifact, and only then creates and pushes the
release tag.

## Phase 17 Outputs

| Area | Output |
|---|---|
| Publish guard | `mcp/samvil_mcp/release_publish.py` |
| CLI | `scripts/publish-verified-release.py` |
| Gate inputs | clean tree, version sync, local release gate, remote release gate, tag existence |
| Tests | pure guard tests + fixture dry-run CLI tests |
| Release | `v3.19.0` |

## Workstreams

### W1 — Publish Guard Core

- [x] Require target branch and clean working tree.
- [x] Require synchronized version files.
- [x] Block existing local or remote release tags.
- [x] Require local release gate pass and remote release gate pass.
- [x] Acceptance: guard blocks dirty tree, existing tag, and remote gate failure.

### W2 — Publisher CLI

- [x] Add dry-run fixture mode for deterministic tests.
- [x] Add live mode that pushes branch, waits for Actions, verifies artifact gate, then tags.
- [x] Use `--no-verify` only for tag push after branch version has already advanced.
- [x] Acceptance: CLI returns 0 only when publish guard passes.

### W3 — Dogfood

- [x] Run publish guard unit tests.
- [x] Run publisher CLI fixture dry-run pass/fail tests.
- [ ] Run live publisher dry-run against latest successful main run.
- [ ] Run default release check runner.
- [ ] Build release evidence bundle.
- [ ] Run full pre-commit.

### W4 — Release

- [ ] Update README/CHANGELOG/version files.
- [ ] Merge to main.
- [ ] Use verified publisher to push main, wait for CI, verify artifact gate, and push `v3.19.0`.

## Non-Goals

- Replacing GitHub branch protection.
- Publishing package registry artifacts.
- Automatically editing version files.

## Release Bar

- Publisher fixture tests pass for pass and blocked remote evidence.
- Local release runner and pre-commit pass at `v3.19.0`.
- Live publisher flow pushes tag only after remote artifact gate passes.
