# Slack 메시지 전송 가이드

이 문서는 Claude Code 등에서 Slack으로 메시지를 보낼 때 따라야 할 규칙을 정리한 것이다.
전송 수단(MCP 플러그인, Webhook, API 직접 호출 등)과 무관하게 항상 적용한다.

---

## 0. Slack MCP 플러그인 설치

Claude Code에서 Slack을 사용하려면 공식 Slack MCP 플러그인이 필요하다.
아직 설치하지 않았다면 아래 방법 중 하나로 설치할 것:

- Claude Code 세션 내: `/plugin install slack`
- CLI에서: `claude plugin install slack`

설치하면 Slack MCP 서버(`https://mcp.slack.com/mcp`)에 자동 연결되고
OAuth 인증 흐름이 시작된다. 워크스페이스 관리자가 MCP 연동을 사전 승인해야 한다.

> 플러그인이 설치되어 있지 않은 상태에서 Slack 전송을 요청받으면,
> 먼저 설치를 안내할 것.

---

## 1. 긴 내용은 제목 + 스레드로 분리

채널 타임라인을 깔끔하게 유지하기 위해, 긴 메시지는 분리해서 보낸다.

**판단 기준:**
- 3줄 이하 → 채널 메시지 하나로 전송
- 4줄 이상 또는 코드 블록/리스트 포함 → 분리

**전송 패턴:**
1. 채널 메시지: 한 줄 요약 (예: `*[배포 완료]* api-server v2.3.1 production`)
2. 스레드 답글: 상세 내용 전체

`reply_broadcast`는 기본 `false`. 채널 전체에 노출이 필요할 때만 사용자 확인 후 `true`.

## 2. mrkdwn 문법 사용 (표준 Markdown 금지)

Slack은 자체 포맷인 mrkdwn을 사용한다. 일반 Markdown을 그대로 보내면 깨진다.

### 핵심 차이 요약

| 의도 | ❌ Markdown | ✅ mrkdwn |
|------|-----------|----------|
| 굵게 | `**text**` | `*text*` |
| 기울임 | `*text*` | `_text_` |
| 취소선 | `~~text~~` | `~text~` |
| 링크 | `[label](url)` | `<url\|label>` |
| 제목 | `# Heading` | `*제목*` (굵게로 대체) |

> 가장 흔한 실수: `*text*`를 기울임으로 쓰는 것. mrkdwn에서 이것은 **굵게**다.

### mrkdwn 전체 문법

```
굵게:        *텍스트*
기울임:      _텍스트_
취소선:      ~텍스트~
인라인코드:  `코드`
코드블록:    ```코드 블록```
링크:        <https://example.com|표시할 텍스트>
인용:        > 인용문
멘션:        <@U012AB3CD>
채널링크:    <#C012AB3CD>
전체알림:    <!channel>  <!here>  <!everyone>
불릿리스트:  • 항목 (또는 - 항목)
번호리스트:  1. 항목
```

### 지원하지 않는 문법 (사용 금지)

아래는 Slack에서 렌더링되지 않고 문자 그대로 표시된다:
- 헤딩 (`#`, `##`, `###`)
- 테이블 (`| col | col |`)
- 인라인 이미지 (`![alt](url)`)
- HTML 태그 (`<b>`, `<br>`)
- 수평선 (`---`, `***`)
- 중첩 서식 (`*_text_*` — 결과 불안정)
- 각주 (`[^1]`)

### 대안

- 제목이 필요하면 → `*제목 텍스트*` (굵게)
- 테이블이 필요하면 → 코드 블록 안에 정렬된 텍스트
- 구분선이 필요하면 → `───────────` (유니코드 가로선)

## 3. 실전 예시

### 짧은 알림 (채널 메시지 하나로 전송)

```
*[리뷰 완료]* PR #142 approved :white_check_mark: — `fix: Redis TTL 조정`
```

### 긴 알림 (채널 제목 + 스레드)

채널 메시지:
```
*[배포 완료]* `api-server` v2.3.1 → production :rocket:
```

스레드 답글:
```
*배포 상세 정보*

• *서비스:* `api-server`
• *버전:* `v2.3.0` → `v2.3.1`
• *환경:* production
• *배포자:* <@U012AB3CD>
• *소요 시간:* 3분 42초

*주요 변경*
• 사용자 인증 로직 리팩터링
• Redis 캐시 TTL 조정 (30min → 15min)

*모니터링:* <https://grafana.internal/d/abc123|Grafana 대시보드>
```
